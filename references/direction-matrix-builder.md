# Direction Matrix Builder

Use this when taste is unclear, the user asks for options, the first direction feels same-looking, or a design needs 3-5 genuinely different directions before implementation.

## Principle

A useful matrix varies the design thesis, not just colors. Each direction should imply a different composition, product object, density, motion role, and emotional job.

## Matrix Axes

Choose 3-5 axes that matter for the brief:

- product stance: operational utility, product-led story, editorial authority, cultural immersion, playful consumer, technical instrument
- dominant form: workbench, canvas, timeline, object portrait, split pane, atlas, scene, console, gallery, data wall
- density: sparse, balanced, dense-with-air, operational dense
- palette logic: restrained neutral, warm/desirable, bold contrast, material-derived, dark-with-purpose
- motion role: none, feedback, reveal, continuity, kinetic signature
- proof style: real workflow, object close-up, before/after, live state, compact metrics, customer context
- interaction model: browse, configure, compare, command, inspect, approve, simulate, compose

Use both a vertical ladder and horizontal directions when taste is unclear:

- vertical ladder: one variable changes in visible steps, such as color intensity, density, motion appetite, or visual risk
- horizontal directions: the design thesis changes, such as operational workbench, editorial proof, immersive scene, technical instrument, or warm consumer product

## Direction Card Shape

Each option should include:

```text
Direction name:
Design thesis:
Best for:
Dominant form:
Layout behavior:
Palette logic:
Type character:
Motion role:
What it refuses:
Risk:
Why choose it:
```

## Boundary Samples

When the user is tuning taste, include boundary samples:

- `safe`: professional but possibly too quiet
- `target`: likely best fit
- `bold`: more memorable but riskier
- `overload`: intentionally too much so the upper bound is visible

Boundary samples help users say "less than that, more than this" without abstract taste vocabulary.

## Good Versus Weak Matrices

Good:

- one option is calm and operational, another is editorial, another is immersive or instrument-like
- each option changes the dominant form
- each option has a clear refusal
- risks are different, not repeated
- the user can choose based on product strategy, not visual taste alone

Weak:

- all options use the same hero layout
- only the accent color changes
- all options rely on cards, glow, and rounded panels
- the names are aesthetic adjectives with no product consequence
- every option could be implemented with the same component tree

## Fusion Rule

If the user likes different parts of different directions, fuse them explicitly:

```text
Use A for palette placement.
Use B for density.
Use C for dominant form.
Reject D's motion because it conflicts with the product register.
```

Do not blend all options into a vague compromise.

## Required Contrast

Before presenting options, check that at least three of these differ between each direction:

- first focal point
- section rhythm
- component density
- dominant object
- palette role structure
- type character
- motion role
- interaction model

If fewer than three differ, the matrix is only a set of variants.

## Selection Logic

Recommend one direction only after explaining fit:

- choose operational utility when repeated work, data, permissions, or error recovery matters
- choose product-led story when the page must sell a workflow quickly
- choose editorial authority when trust, expertise, or premium clarity matters
- choose cultural immersion when the subject has strong native material
- choose technical instrument when the product is about control, measurement, simulation, or inspection
- choose playful consumer when delight, habit, and approachability matter more than dense control

## Compression Rule

After the user picks one direction, collapse the others. Do not keep mixing them unless the user explicitly asks for a hybrid.
